## 项目简介

专为参加创新创业比赛的大学生团队设计的 AI 辅助写作工具集，采用**六阶段工作流**：

```
项目主档 → Research → 分析稿 → 成稿 → Review修订 → 统稿可视化
```

**核心能力**：
- 项目主档作为唯一信息源，状态标注（✅已确认 / ⚠️待补充 / 🔍待验证）
- 三路可视化分流（Python数据图 / HTML结构图 / MiniMax概念图）
- 两阶段输出流程（正文先行，图表后置）
- 答辩级视觉规范（禁止AI审美）

**适用比赛**：大创、挑战杯、创青春、互联网+、各类商赛

## 六阶段工作流

### 阶段0：建立项目主档
**触发**：「建立项目」/「新建项目」
**输出**：Project Master（唯一信息源）

### 阶段1：Research补证据
**触发**：「搜索市场」/「查竞品」
**输出**：证据池（数据/政策/竞品/案例）
**注意**：只补公开信息，不补私有事实

### 阶段2：分析稿模式
**触发**：「写第X章」/「开始写作」
**输出**：结构化版本（章节结构/关键判断/数据点/缺失项/可视化建议）

### 阶段3：成稿模式
**触发**：「第X章全部输出」
**输出**：正式比赛文书（连续正文/多维论证/克制文风）

### 阶段4：Review与修订
**触发**：「帮我检查」/「review」
**输出**：修订稿（主编协调交叉review）

### 阶段5：统稿与可视化输出
**触发**：「统稿」/「最终版」
**输出**：定稿 + 可视化图表

## 两阶段输出流程（默认）

当你说「第X章全部输出」时，默认执行两阶段：

### 阶段一：正文先行
1. 读取 `.project-config.md` 获取项目路径
2. 输出成稿正文，正文中使用**图位占位符**：`图4-1【待生成】：xxx`
3. 同时输出可视化清单，标注优先级：
   - **必做**：没有它正文明显变弱
   - **建议做**：有了更好，但没它也能交
   - **可选**：主要用于答辩增强
4. 等待你确认清单

### 阶段二：批量生成
5. 你确认后，一次性生成全部图表
6. 输出最终汇总包

### 核心原则
- 正文逻辑先于图表生成
- 图表必须服务正文，不允许脱离正文单独堆图
- 如果某张图不如文字清晰，保留文字，不强制生成

## 核心规则

1. **信息源唯一** - 所有章节从主档提取，禁止编造主档没有的信息
2. **硬性缺失停止** - 市场规模/竞品/团队/财务等缺失必须补充后才能推进
3. **正文先于图表** - 图位占位符在正文阶段就写入，图表在确认清单后生成
4. **克制文风** - 禁止宣传腔，禁止"颠覆/革命性/绝对领先"等词
