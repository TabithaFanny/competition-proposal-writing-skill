> 基于 AI 的大学生创新创业比赛计划书写作助手，采用"项目主档→框架生成→逐章深写→全稿统一"四阶段架构

## 项目简介

这是一个专为参加创新创业比赛的大学生团队设计的 AI 辅助写作工具集，采用**四阶段工作流**：

1. **项目主档建立** - 建立唯一信息源
2. **12章框架生成** - 钉住每章写什么
3. **逐章深写** - 单章独立+继承上下文
4. **全稿统一** - 口径/风格/数据一致

**v2.0 核心升级**：
- 项目主档作为唯一信息源
- 状态标签机制（已确认/待补充/待验证/AI推断）
- 每章附带PPT提炼+配图关键词+缺失证据清单
- 模块化reference文件，易维护可扩展

**适用比赛**: 大创、挑战杯、创青春、互联网+、各类商赛

## 四阶段工作流

### 阶段一：项目主档建立
**触发**: 提供项目资料 → 输出结构化主档

```
输入: 项目名称、核心痛点、目标用户、核心功能...
      ↓
处理: 信息抽取 → 状态标注
      ↓
输出: 项目主档（含40+字段，状态标注）
```

### 阶段二：12章框架生成
**触发**: 主档建立后 → 输出骨架

```
输入: 项目主档
      ↓
处理: 映射到12章结构 → 标注来源
      ↓
输出: 12章骨架（含每节信息来源+缺失证据清单）
```

### 阶段三：逐章深写
**触发**: 指定章节 → 输出完整章节

```
输入: 主档+章节号
      ↓
处理: 继承上下文 → 标出缺失 → 写作
      ↓
输出: 正式章节内容 + PPT提炼 + 配图关键词 + 缺失清单
```

### 阶段四：全稿统一
**触发**: "统一校对" → 输出修订版

```
输入: 所有已写章节
      ↓
处理: 术语统一 → 数据一致性 → 风格检查
      ↓
输出: 统一报告 + 修改后完整版本
```

## 使用示例

**启动新项目**：
```
请帮我建立项目主档：
项目是一个AI简历优化系统，核心创新是NLP技术，
目标用户是应届生，团队3人...
```

**生成框架**：
```
主档已建立，请生成12章框架
```

**深写某一章**：
```
请帮我写第三章市场分析
```

**统一校对**：
```
请帮我统一校对全文
```

**PPT配图**：
```
请帮我生成第5页市场分析的配图关键词
```

## 核心规则

1. **信息源唯一** - 所有章节从主档提取，禁止编造
2. **硬性缺失停止** - 市场规模/竞品/团队/财务等缺失必须补充
3. **软性增强可写** - 用户案例/试点反馈等缺失可AI推断填充
4. **输出Word友好** - 正文纯文本，可直接复制到Word
