# 表格使用规则 v1.0

> 核心原则：少用表格，多用语言论证。表格是信息密度最高的表达方式，但也是最容易被"跳过"的方式。

---

## 表格过度使用的问题

### 问题1：评委不读表

评委读计划书的方式是"扫读+跳读"：
- 扫读：读段落开头句
- 跳读：找数字看结论
- **不读**：大表格、小字号、多行列

### 问题2：表格割裂论证

表格把论证切成碎片，评委无法连贯理解判断的逻辑链条。

### 问题3：表格≠论证

表格展示"是什么"，不展示"为什么"。比赛计划书的核心是论证，不是清单。

---

## 什么时候可以用表格

### 允许使用表格的场景

| 场景 | 示例 | 理由 |
|------|------|------|
| 需要精确对比 | 竞品功能矩阵 | 横向对比需要精确位置 |
| 数据结构展示 | 财务科目拆解 | 一级科目、二级科目需要对应 |
| 参考信息罗列 | 政策文件清单 | 信息本身就是清单性质 |
| 枚举但不解释 | 团队成员分工 | 分工本身不需要论证 |

### 禁止使用表格的场景

| 场景 | 替代方案 |
|------|----------|
| 市场规模假设 | 用文字+漏斗图 |
| 商业模式论证 | 用文字+流程图 |
| 竞争优势说明 | 用文字+象限图 |
| 发展历程/里程碑 | 用文字+时间轴 |
| 用户画像描述 | 用文字+卡片 |

## 表格使用标准（允许情况下）

### 最小表格标准

```
✅ 可以：
- 2-3行、2-4列
- 评委3秒能看清结构
- 配合简短文字说明

❌ 不可以：
- 超过5行或5列
- 字号小于10pt
- 有合并单元格
- 表头超过2行
```

### 表格必须有的元素

1. **标题**：一句话说明这是什么表
2. **来源说明**：数据来源或编制依据
3. **简短解读**：表中1-2个关键发现
4. **注释**：如有特殊情况必须说明

## 论证替代表格示例

### 示例1：市场规模

**表格写法（不推荐）**：
```
| 指标 | 数值 |
|------|------|
| TAM | 600亿 |
| SAM | 3亿 |
| SOM | 30万 |
```

**文字写法（推荐）**：
```
全国社区治理数字化市场呈现金字塔结构：底层是包含10万个社区的潜在市场（TAM），按年均5万元数字化投入计算，约600亿元；中间层是排除数字化基础薄弱、支付意愿不足的社区后的可服务市场（SAM），约3亿元；顶层是第三年实际可覆盖的可获得市场（SOM），约30万元。这种层层筛选的结构，决定了我们的市场进入策略必须从小切口起步。
```

### 示例2：竞品对比

**表格写法（可接受）**：
```
| 维度 | 本项目 | 竞品A | 竞品B |
|------|--------|-------|-------|
| AI前置识别 | ✓ | ✗ | ✗ |
| 闭环服务 | ✓ | ✓ | ✗ |
| 人机协同 | ✓ | ✗ | ✓ |
```

**文字写法（推荐）**：
```
本项目与市场现有方案的核心差异在于"前置识别+闭环服务"的完整机制。竞品A具备基础的闭环服务能力，但缺少AI前置识别环节；竞品B虽引入AI辅助，但未形成从识别到处置的完整闭环。本项目通过规则引擎与大模型的混合架构，实现了从居民诉求表达到问题解决的全链路覆盖，这是现有竞品尚未突破的能力边界。
```

## 表格密度自检

```
完成章节后检查：
□ 全章表格数量是否 ≤ 3个？
□ 每个表格是否在2-4列、2-5行范围内？
□ 每个表格是否有标题、来源、解读、注释？
□ 表格是否可以被对应的文字段落替代？（如果可以，删除表格）
□ 评委看表格时是否能3秒理解结构？
```

## 通用规则

```
可用表格的情况：
□ 竞品功能矩阵（精确对比需要）
□ 财务科目拆解（一级二级对应）
□ 政策文件清单（本身就是清单）
□ 团队分工（分工不需要论证）
□ 数据来源引用（需要精确位置）

不可以用表格的情况：
□ 论证判断（"为什么选A"）
□ 市场规模（用漏斗图）
□ 商业模式（用流程图）
□ 竞争优势（用象限图）
□ 用户画像（用文字+场景描述）
```

---

## 与其他规范的衔接

```
章节写作 → 发现信息罗列需求
     │
     ▼
判断：这是"需要论证"还是"需要展示"？

如果是"需要论证"：
→ 不用表格
→ 用文字展开5层论证
→ 图表用结构型（HTML）

如果是"需要展示"（清单/对比/拆解）：
→ 可用表格
→ 但控制行列数
→ 配合文字解读
```
