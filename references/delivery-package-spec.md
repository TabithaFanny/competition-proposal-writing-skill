# 章节交付包规范 v1.0

> 核心原则：每章形成完整交付闭环。写完正文不等于完成，还必须有配套的图表、说明、状态记录。

---

## 交付包定义

每完成一章，必须同时交付以下内容：

```
chapter-{N}/
├── 01_正文.md                    # 成稿正文（必须）
├── 02_可视化需求.md             # 图表清单+优先级（必须）
├── 03_图表/                      # 图表文件目录（按需）
│   ├── *.html                   # HTML结构图
│   └── *.png                   # Python数据图
├── 04_review.md                 # 本章Review记录（必须）
└── 05_状态.json                 # 完成状态标记（建议）
```

---

## 各组件说明

### 01_正文.md（必须）

**命名**：必须是这个精确名称
**内容**：成稿正文，含图位占位符
**图位占位符格式**：
```
图{N}-{M}【待生成】：{图表名称}
图6-1【待生成】：三阶段发展战略图
图6-2【待生成】：组织架构图
```

### 02_可视化需求.md（必须）

**命名**：必须是这个精确名称
**内容**：
```markdown
## 本章可视化需求

| 位置 | 图表名称 | 类型 | 优先级 | 核心信息 | 状态 |
|------|----------|------|--------|----------|------|
| 6.1.2 | 三阶段发展战略图 | HTML | 必做 | 试点→区域复制→跨区域推广 | 待生成 |
| 6.3.1 | 组织架构图 | HTML | 必做 | 四部门+项目负责人 | 待生成 |

### 生成顺序
1. 优先生成"必做"级别图表
2. 建议做图表在时间允许时生成
3. 可选图表默认跳过

### 答辩增强建议
（如时间允许，可额外生成）
- 暂无
```

### 03_图表/（按需创建）

**命名**：必须是这个目录名
**内容**：本章所有图表文件
**文件命名**：
```
图{N}-{M}_{图表名称}.html
图6-1_三阶段发展战略图.html
图6-2_组织架构图.html
```

### 04_review.md（必须）

**命名**：必须是这个精确名称
**内容**：本章的Review记录
```markdown
## 第六章 Review记录

### 完成度
- 字数：~8,022字
- 完成度：~80%
- 状态：✅ 初稿完成

### Review结论
- [Reviewer1]: 第三章可补充更多实施细节
- [Reviewer2]: 建议增加阶段性里程碑说明

### 待处理
- [ ] 扩充6.1.3实施路径
- [ ] 补充6.2.2成本结构详解
```

## 交付完成标准

### 检查清单

```
完成本章后，必须确认：

□ 01_正文.md 存在且完整
□ 02_可视化需求.md 存在且包含完整清单
□ 04_review.md 存在且有Review记录
□ 正文中图位占位符与02_可视化需求.md中清单一致
□ 图表文件存放在 03_图表/ 目录
□ 图表文件命名格式正确
□ 字数达标（按chapter-depth-guide要求）
```

### 交付质量标准

```
正文质量：
□ 无跳跃论证（每判断有证据，每证据有解释）
□ 无表格堆砌（≤3个表格）
□ 无宣传腔（无人"令人振奋"等）
□ 字数密度达标

图表质量：
□ 每张图评委3秒能看懂核心信息
□ 无量级差异误导（分层展示）
□ 视觉中心突出

Review质量：
□ 至少记录1轮Review
□ 有明确的待处理项
□ 已处理项有标记
```

## 交付包与Review流程的衔接

```
章节写作完成
     │
     ▼
生成交付包（01-04）
     │
     ▼
触发Review
     │
     ├── Review通过 → 标记完成，进入下一章
     │
     └── Review不通过 → 修订正文 → 更新交付包 → 再次Review
```

## 常见问题

### Q: 图没有生成怎么办？

A: 正文必须带图位占位符，不是空着。占位符格式：`图{N}-{M}【待生成】：{图表名称}`

### Q: Review发现大问题怎么办？

A: 大问题进入修订循环，小问题可记录待后续处理。但必须保证交付包中的正文是最新修订版本。

### Q: 图表生成后如何更新？

A: 生成图表后：
1. 将图表文件放入 `03_图表/` 目录
2. 更新 `02_可视化需求.md` 中的状态
3. 将正文中的 `【待生成】` 替换为实际文件名
4. 在 `04_review.md` 中记录图表生成时间
