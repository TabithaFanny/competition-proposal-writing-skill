# 可视化质量分级标准 / Visual Quality Guide

> 明确什么图叫"分析级"、什么叫"文档级"、什么叫"答辩级"。

---

## 质量分级定义

### Level 1：分析级
内部使用，重点是把信息表达清楚。
- 允许偏朴素
- 允许保留较多细节
- 不需要视觉设计

### Level 2：文档级
可放项目书，要求整洁、清楚、有统一样式。
- 配色统一
- 标签清晰
- 信息不过载
- 有基本视觉层次

### Level 3：答辩级（目标标准）
可直接放PPT，要求：
- **3秒能看懂** - 评委第一眼就知道你想说什么
- **有视觉中心** - 只有一个主焦点
- **有主次** - 本项目/核心结论最突出
- **信息不过载** - 默认不超过5个视觉对象
- **结论先于细节** - 核心结论直接可见

---

## 答辩级图表核心规则

### 规则1：一张图只表达一个核心结论

**禁止：**
- 把多个无关信息堆在一张图里
- 想在一张图里说完所有事

**正确：**
- 每张图只有一个主角
- 副信息服务于主角

### 规则2：默认不超过5个视觉对象

**超过5个时的处理：**
- 删减非核心信息
- 合并相似项
- 拆分成多张图

### 规则3：优先突出"本项目"

- 本项目用最强视觉（最大色块/最深颜色/最高位置）
- 竞品/对比项用中性色
- 评委第一眼看到的就是你想强调的

### 规则4：图中说明文字不超过两层

**第一层：** 核心标题（一句话）
**第二层：** 关键注释（补充说明）
**禁止：** 大段解释性文字

### 规则5：PPT友好原则

- 优先横向布局（适配投影）
- 留白充足（不拥挤）
- 字体可读（最小12pt）
- 颜色对比清晰

## 自动降噪规则

系统在出图前自动执行以下判断：

### 降噪1：元素过多时自动删减

```
判断条件：视觉对象 > 5个
处理方式：
1. 保留：本项目 + 核心对比项（≤3个）
2. 弱化：其他项用小标签或省略
3. 拆分：如无法删减，拆成多张图
```

### 降噪2：数值量级差过大时禁止共轴

```
判断条件：最大值 / 最小值 > 100
禁止：直接用同一Y轴柱状图（会误导）

正确做法（选一）：
A. 分层漏斗/分层卡片：每层独立比例
B. 对数坐标：展示量级关系
C. 分段说明：用文字说明筛选逻辑而非柱高
```

### 降噪3：标签过长时切换图型

```
判断条件：标签字数 > 10字
处理方式：
A. 改用卡片/注释形式
B. 缩写标签，完整说明放图例
C. 改用结构图代替标签
```

### 降噪4：图不如文字清晰时禁止强制做图

```
判断条件：图的信息密度 > 文字信息密度
判断标准：
- 数据点 < 3个
- 关系简单无需图示
- 图反而需要更多解释

处理方式：不画图，直接用文字说明
```

## TAM/SAM/SOM专用规则

### 问题：量级差异过大

```
TAM = 6800亿元
SAM = 3亿元
SOM = 30万元

6800亿 vs 30万 = 2,266,667倍
直接共轴柱状图会让后两者"消失"
```

### 推荐方案：分层漏斗图

```
结构：
┌─────────────────────────────────┐
│          TAM: 6800亿              │  ← 全国智慧社区市场
│   （标注：为什么这么大）            │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│          SAM: 3亿元              │  ← 筛选条件
│   （标注：哪三重筛选）             │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│          SOM: 30万元            │  ← 三年20社区
│   （标注：如何获得）               │
└─────────────────────────────────┘

核心表达：不是柱高对比，而是筛选逻辑
```

### 禁止做法

- ❌ 同一坐标轴柱状图（量级差太大）
- ❌ 堆叠柱状图试图展示层级
- ❌ 用对数坐标假装解决了问题

## 竞品对比图专用规则

### 分析版（放正文）

```
可以保留：
- 5个竞品全列
- 4个维度全列
- 评分数值

要求：
- 表格清晰
- 颜色区分本项目
```

### 答辩版（放PPT）

```
只保留：
- 关键竞品（≤3个）
- 关键维度（≤3个）
- 本项目优势点（≤3个突出优势）

突出：
- 本项目 vs 传统方案的差异
- 评委最关心的1-2个维度
```

## 市场定位图专用规则

### 关系图 vs 卡片排版

**卡片排版（错误）：**
```
┌─────┐   ┌─────┐   ┌─────┐
│  A  │   │  B  │   │  C  │
└─────┘   └─────┘   └─────┘
```
只是并列，没有关系

**关系图（正确）：**
```
           A
          /|\
         / | \
        /  |  \
       /   |   \
      B────C────D

边表示关系：
- A→B：转化路径
- B→C：协同机制
- C→D：闭环结构
```

### 三角定位图结构

```
顶点：
A = 前置识别
B = 人机协同
C = 闭环服务

边：
A→B：识别后分流
B→C：处理后沉淀
C→A：治理反哺识别

中心：人机协同（核心价值）
```

## 输出质量检查清单

生成图表后，必须逐项检查：

```
□ 评委3秒内能说出这张图想表达什么
□ 视觉中心是本项目/核心结论
□ 没有超过5个主要视觉对象
□ 没有量级差异导致的误导
□ 标签/说明不超过两层
□ 横向布局，PPT友好
□ 留白充足，不拥挤
□ 颜色对比清晰可读
```

## 与skill的衔接

```
章节写作 → 发现可视化需求
     │
     ▼
系统判断类型：
     │
     ├── 数据型 → Python
     │        → 按visualquality-guide检查
     │        → 禁止量级差异共轴
     │
     ├── 结构型 → HTML
     │        → 按visualquality-guide检查
     │        → 必须是关系图而非卡片
     │
     └── 概念型 → MiniMax图片
              → 按visualquality-guide检查
```

## 速查

| 问题 | 解决方案 |
|------|----------|
| 量级差异大 | 分层漏斗/分层卡片 |
| 元素过多 | 删减到≤5个 |
| 标签过长 | 改卡片/缩写 |
| 只是卡片排版 | 改成关系图 |
| 图不如文字清晰 | 不画图 |
