# 单章深写 Prompt 规范

> 当用户指定某一章深入写作时，使用本规范。

---

## 单章深写四步法

### 第一步：确认字数要求
在开始写之前，先确认本章的字数要求：

```
本章最低字数：8000字
字数不足70%的，视为不合格，需扩写
扩写不是堆字数，是补论证（判断→证据→解释→影响→结论）
```

### 第二步：继承上下文
在开始写之前，先回显已从主档继承的信息：

```
已继承的项目主档信息：
- 项目名称：[名称]
- 核心痛点：[痛点1、痛点2]
- 目标用户：[用户画像]
- 核心功能：[功能1、功能2、功能3]
- 技术方案：[技术栈]
- 市场规模：[TAM/SAM/SOM]
- 竞品信息：[竞品A、竞品B]
- 商业模式：[盈利方式]
- 团队信息：[创始人、联合创始人...]
```

### 第三步：标出缺失变量
列出本章写作需要的但主档中没有的信息：

```
缺失变量：
1. [变量名] - 用于[用途] - 状态：⚠️待补充
2. [变量名] - 用于[用途] - 状态：🔍待验证
```

### 第四步：检查信息来源
根据本章来源标注（见framework-generation.md），确认可以引用哪些主档字段。

### 第五步：输出完整章节
按照framework-generation.md中本章的结构和内容要求，输出完整章节内容。

---

## 通用输出格式

每章深写完成后，统一输出以下内容：

### A. 正式章节内容
按照框架结构输出完整内容，注意：
- Word友好格式（无多余markdown符号）
- 一级标题 → 二级标题 → 三级标题 → 正文
- 表格用标准markdown表格格式
- 避免套话，用数据说话

### B. 本章PPT提炼
```
## 本章PPT提炼
- 页面标题：[标题]
- 核心结论：[一句话]
- 3个要点：
  [1] [要点1]
  [2] [要点2]
  [3] [要点3]
- 配图关键词：[英文关键词，逗号分隔]
- 图表建议：[图表类型]
```

### C. 缺失证据清单
```
## 缺失证据清单
- □ [证据项1]
- □ [证据项2]
```

## 章节引用规则

| 章节 | 允许引用 | 必须补充 | 禁止编造 |
|------|----------|----------|----------|
| 第一章执行摘要 | 全主档 | 当前进展、融资需求 | 数据必须验证 |
| 第二章背景 | 主档-痛点 | 行业数据 | 不能夸大 |
| 第三章产品 | 主档-功能 | 产品细节 | 技术参数必须准确 |
| 第四章市场 | 主档-用户、竞品 | 市场规模 | 数据必须可查 |
| 第五章商业 | 主档-定位 | 收入测算 | 财务假设必须合理 |
| 第六章营销 | 竞品、产品 | 定价、渠道 | 成本必须可查 |
| 第七章团队 | 主档-团队 | 成员细节 | 履历必须属实 |
| 第八章运营 | 主档-进展 | 数据佐证 | 数字必须可验证 |
| 第九章财务 | 商业+营销 | 详细测算 | 必须有依据 |
| 第十章风险 | 技术+商业 | 应对策略 | 不能回避问题 |
| 第十一章社会 | 主档-定位 | 量化影响 | 不能夸大 |
| 第十二章附录 | 全部 | 支撑材料 | 材料必须真实 |

## 状态回退规则

当缺失关键信息时：

### 硬性缺失（P0）→ 停止写作，询问用户
```
⚠️ 无法继续写作，因为缺少以下关键信息：
1. [缺失项] - 本章[具体用途]必须
请补充后重新请求写作。
```

### 软性增强（P1）→ 标注后继续，用AI推断填充
```
📝 软性增强：[缺失项]暂无，AI基于[依据]推断为[内容]。
请确认是否准确，或补充真实信息。
```

### 待验证（🔍）→ 标注数据来源，继续写作
```
📊 待验证：[数据]引用自[来源]，请确认该数据准确性。
如无可靠来源，建议标注"待验证"或删除该数据点。
```

## 风格规范

### 正面示范（推荐）
- "根据《中国家庭教育白皮书》，2023年家庭教育支出同比增长15%"
- "我们的核心功能——AI智能错题本——通过自然语言处理技术..."

### 负面示范（避免）
- "我们的产品非常先进，能够帮助用户..."
- "市场潜力巨大，前景广阔..."

### 量化原则
- 能用数字就不用模糊表达
- 数字要标注来源或估算方法
- 百分比要给出绝对值参考

### 评委友好原则
- 结论先行（先说结论，再说依据）
- 重点加粗（用**加粗**标注关键词）
- 层次清晰（使用三级以内标题）
