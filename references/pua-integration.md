## 持续工作模式

> PUA 的核心价值：**不让 agent 停下来等用户**，遇到问题自己搜答案。

### 决策流程（不停止）

```
发现问题/疑问
    │
    ▼
PUA 追问："数据在哪？依据是什么？"
    │
    ▼
自动 WebSearch 搜索答案
    │
    ▼
基于搜索结果做决定
    │
    ▼
继续写，不等用户
```

### 示例场景

**场景1：这句表述是否太绝对？**
```
PUA："'最优秀'这个词，你验证过吗？"
  → 自动搜索：竞品对比数据
  → 改为可量化表述或删除
```

**场景2：这段要不要删？**
```
PUA："这段的论证贡献是什么？没有数据支撑就删。"
  → 自问：这段有数据吗？有论证吗？
  → 有：保留并补充来源
  → 无：直接删
```

**场景3：竞品分析够完整吗？**
```
PUA："才2个竞品，你的竞品视野不够。"
  → 自动搜索主要竞品
  → 补充到3-5个
```

## 何时触发 PUA

### Research 阶段（推荐高强度使用）

| 场景 | 触发条件 | PUA 做什么 |
|------|----------|------------|
| 市场数据搜索 | 搜索结果少于3条 | 继续搜索，不停止 |
| 政策文件查找 | 未找到最新文件 | 多关键词重试 |
| 竞品分析 | 竞品数量少于2个 | 深挖竞品官网/报道 |
| 行业数据 | 数据过期或不精确 | 寻找多个数据源交叉验证 |

**Research PUA 触发语**："/pua" 或 "继续搜，还没穷尽"

### 写作阶段

| 场景 | 触发条件 | PUA 做什么 |
|------|----------|------------|
| 章节初稿 | 连续2次修改后仍不满意 | 升级味道，换角度重写 |
| 论据不足 | 论证被Review指出缺乏数据 | 强制返回Research补充 |
| 内容空洞 | 文字过于泛泛，没有数据支撑 | PUA 追问"数据在哪" |

**写作 PUA 触发语**："/pua 写作模式" 或 "你又在原地打转"

### Review 阶段

| 场景 | 触发条件 | PUA 做什么 |
|------|----------|------------|
| 自我Review | 写完后自己检查 | 用 PUA 挑剔视角审视 |
| 修订反馈 | 主编提出修改意见 | 不找借口，主动承担 |

## PUA 工作流集成

### 阶段1：Research + PUA

```
用户：帮我搜索社区社工市场规模
  │
  ▼
AI 执行搜索 → 结果<3条
  │
  ▼
触发 PUA L1 → "你这个数据量不够，还有什么渠道？"
  │
  ▼
继续搜索（政府统计局/行业协会/学术论文）
  │
  ▼
结果≥3条 或 PUA L3 → 停止
```

### 阶段2：写作 + PUA

```
用户：写第四章市场分析
  │
  ▼
AI 初稿 → Review 指出数据不足
  │
  ▼
触发 PUA → "你的数据支撑在哪？没有数据的结论叫自嗨"
  │
  ▼
返回 Research 补充 → 新数据
  │
  ▼
重新写章节 → 完成
```

### 阶段3：Review + PUA

```
用户：帮我检查已写的章节
  │
  ▼
激活 PUA Review 模式
  │
  ▼
PUA 视角检查：
  - 数据有来源吗？
  - 论证有漏洞吗？
  - 还有什么没想到？
  │
  ▼
输出 Review 报告 + 修改建议
```

## PUA 自检清单（写作场景）

- [ ] 市场数据有2个以上独立来源吗？
- [ ] 竞品分析覆盖了主要玩家吗？
- [ ] 政策文件是最新的吗？（近1年内）
- [ ] 财务预测有合理假设支撑吗？
- [ ] 团队介绍有具体事例吗？
- [ ] 社会价值有量化指标吗？

---

## 注意事项

1. **PUA 不替代 Research** — PUA 是驱动力，但数据必须真实
2. **禁止编造数据** — PUA 逼你穷尽搜索，但不逼你造假
3. **味道选择** — 阿里味适合严谨论证，字节味适合快速迭代
4. **保持专业克制** — PUA 风格影响语气，不影响内容的学术严谨性
