# 引用与命名规范 v1.0

> 核心原则：图表编号、文件命名、正文引用、数据来源必须统一、可追溯。

---

## 一、图表编号规则

### 编号格式

```
图{章节号}-{序号}
```

### 示例

```
图6-1    ← 第六章第1张图
图6-2    ← 第六章第2张图
图9-1    ← 第九章第1张图
```

### 编号原则

1. **按章节顺序**：先出现的图编号小
2. **不重复**：同一章节内序号连续
3. **不断号**：不在编号中留空
4. **不跨章**：不出现"图5-3"在第6章的情况

### 编号分配记录

在可视化需求文档中记录：
```markdown
| 位置 | 图表名称 | 编号 | 状态 |
|------|----------|------|------|
| 6.1.2 | 三阶段发展战略图 | 图6-1 | 待生成 |
| 6.3.1 | 组织架构图 | 图6-2 | 待生成 |
```

## 二、正文引用规则

### 引用格式

```
如图6-1所示，……

如图6-1、图6-2所示，……

（详见图6-1）
```

### 引用位置

1. **首次出现时引用**：图第一次在正文中提及时必须引用
2. **解释时引用**：解释图表内容时必须引用
3. **结论时引用**：从图表得出结论时必须引用

### 引用与图表生成的对应

```
正文写作 → 使用占位符 → 图6-1【待生成】
     │
     ▼
图表生成完成 → 正文替换占位符 → 图6-1（三阶段发展战略图）
     │
     ▼
图表文件：图6-1_三阶段发展战略图.html
```

## 三、文件命名规则

### HTML结构图

```
图{章节号}-{序号}_{图表中文名}.html
```

**示例**：
```
图6-1_三阶段发展战略图.html
图6-2_组织架构图.html
图3-1_系统分层架构图.html
```

### Python数据图

```
图{章节号}-{序号}_{图表中文名}.png
```

**示例**：
```
图2-1_TAM-SAM-SOM市场规模图.png
图8-1_三年收入预测图.png
```

### Python生成脚本

```
图{章节号}-{序号}_{图表中文名}.py
```

**示例**：
```
图2-1_TAM-SAM-SOM市场规模图.py
图8-1_三年收入预测图.py
```

### 低保真文档

```
图{章节号}-{序号}_{图表中文名}-LowFi.md
```

**示例**：
```
图6-1_三阶段发展战略图-LowFi.md
```

## 四、数据来源标注规则

### 标注位置

数据来源紧跟在数据后面，用括号标注。

### 标注格式

```
{数据内容}（来源：{来源类型}）

如：300份（来源：项目调研）
如：600亿元（来源：艾瑞咨询《2024年中国社区治理数字化报告》）
```

### 来源类型分类

| 类型 | 标注方式 | 示例 |
|------|----------|------|
| 项目调研 | （来源：项目调研） | 300份问卷 |
| 公开报告 | （来源：艾瑞咨询《2024年报告》） | 行业数据 |
| 政府文件 | （来源：民政部《XXX通知》） | 政策数据 |
| 学术论文 | （来源：《社会学研究》2023年第X期） | 研究数据 |
| 竞品公开 | （来源：竞品官网/公开材料） | 竞品信息 |
| 估算推演 | （来源：项目组测算） | 财务预测 |

### 禁止的来源标注

```
❌ （来源：网络）
❌ （来源：网上查的）
❌ （来源：据说）
❌ （来源：大概）
```

## 五、图表清单文件规范

### 文件位置

每章目录下必须有图表清单文件，命名：`04_图表清单.md`

### 清单格式

```markdown

| 编号 | 图表名称 | 类型 | 存放路径 | 状态 |

## 图号与正文对应
```

## 六、章节目录结构规范

### 完整结构

```
chapter06-运行保障/
├── 01_正文.md                    # 成稿正文
├── 02_可视化需求.md             # 可视化清单
├── 03_图表/                      # 图表目录
│   ├── 图6-1_三阶段发展战略图.html
│   └── 图6-2_组织架构图.html
├── 04_图表清单.md                # 图表清单
└── 04_review.md                  # Review记录
```

### 文件命名检查

```
□ 正文文件：01_正文.md（不是"正文.md"）
□ 可视化需求：02_可视化需求.md（不是"图表需求.md"）
□ 图表目录：03_图表/（不是"assets"）
□ 图表清单：04_图表清单.md（不是"清单.md"）
□ Review记录：04_review.md（不是"review.md"）
```

## 七、统稿时的命名统一

### 统稿前检查

```
□ 全文图号无重复
□ 图号与章节对应正确
□ 正文引用与实际图表文件对应
□ 文件命名格式统一
□ 数据来源标注一致
```

### 统稿常见问题

| 问题 | 解决方法 |
|------|----------|
| 图号重复 | 重新编号，确保连续 |
| 正文引用与文件不对应 | 统一替换 |
| 数据来源标注不一致 | 按统一格式替换 |
| 文件命名混乱 | 按规范重命名 |

## 八、与visualquality-guide的衔接

```
正文写作 → 发现可视化需求
     │
     ▼
判断图表类型（HTML/Python）
     │
     ▼
生成图表 → 按命名规范命名
     │
     ▼
记录到图表清单
     │
     ▼
正文引用时按图号引用
```
