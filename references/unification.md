# 全稿统一校对规范

> 当用户说"帮我统一校对"或"润色全文"时，使用本规范。

---

## 统一校对四步法

### 第一步：收集全文
收集所有已写的章节内容，建立完整文档结构。

### 第二步：术语统一检查
建立术语表，检查并统一全文术语使用。

### 第三步：数据一致性检查
交叉验证各章节引用的数据是否一致。

### 第四步：风格统一调整
检查并统一各章节的写作风格、语气、格式。

---

## 术语统一检查

### 必查术语
| 术语 | 正确写法 | 需替换 |
|------|----------|--------|
| 项目名称 | [主档中的名称] | 不可替换 |
| 目标用户 | [统一描述] | 不能一会儿"用户"一会儿"客户" |
| 核心功能 | [统一描述] | 不能名称不一致 |
| 市场数据 | [统一数字] | 不能各章数据矛盾 |

### 输出格式
```
## 术语统一报告
| 术语 | 正确写法 | 出现位置 | 状态 |
|------|----------|----------|------|
| 用户 | [统一] | 第2、3、5章 | ✅已统一 |
| 目标市场 | [统一] | 第4章写X，第5章写Y | ❌矛盾 |
```

---

## 数据一致性检查

### 必查数据
- 市场规模（TAM/SAM/SOM）
- 用户规模/用户数
- 收入预测数字
- 团队成员数量
- 融资金额
- 股权比例

### 输出格式
```
## 数据一致性报告
| 数据项 | 值 | 出现章节 | 状态 |
|--------|----|----------|------|
| TAM | X亿 | 第4、9章 | ✅一致 |
| 用户数 | Y万 | 第2章写Z万，第8章写W万 | ❌矛盾 |
```

---

## 风格统一检查

### 检查项
1. **开头风格**：各章是否都结论先行
2. **量化程度**：是否都用数据说话
3. **套话程度**：是否还有空洞表述
4. **语气一致**：是否都是专业但有温度的语气

### 常见问题
| 问题 | 示例 | 修正建议 |
|------|------|----------|
| 空洞套话 | "前景广阔，市场潜力巨大" | 删除或改为具体数据 |
| 自夸过度 | "最先进、最优秀、绝对领先" | 改为可量化表述 |
| 语气不一 | 一会儿"我们"一会儿"项目组" | 统一为"我们" |

### 输出格式
```
## 风格问题报告
| 位置 | 问题 | 修正建议 |
|------|------|----------|
| 第2章第3段 | [空洞表述] | [具体化建议] |
| 第5章第1段 | [语气不一致] | [统一为"我们"] |
```

---

## 输出格式

### 最终报告
```
## 全稿统一报告

### 1. 术语统一
[术语统一报告]

### 2. 数据一致性
[数据一致性报告]

### 3. 风格问题
[风格问题报告]

### 4. 修改建议
建议按以下顺序修改：
1. [优先级最高]
2. [次高]
3. [可选]
```

### 修改后版本
提供修改后的完整版本，标注修改点。
