# 12章框架模板

> 完整计划书框架，基于项目主档生成。每章附带PPT提炼和缺失证据清单。

---

## 结构规范

- 一级标题：第X章 章节名称
- 二级标题：X.1 小节名称
- 三级标题：X.1.1 子小节名称

## 每节标注规则

在每节后标注信息来源：
- 📌 主档：[字段名] - 表示引用项目主档
- ⚠️ 需补充：[缺失项] - 表示需要用户补充
- 📊 需数据：[数据要求] - 表示需要数据支撑

## 第一章 执行摘要

### 本章目标
一句话概括项目精髓，让评委快速了解项目价值。

### 内容结构
```
执行摘要（300-500字）
  (1) 项目痛点：具体问题+数据佐证
  (2) 解决方案：一句话概括产品/技术/模式
  (3) 核心优势：技术壁垒/资源/团队优势
  (4) 市场潜力：规模+增长+定位
  (5) 商业模式：如何盈利
  (6) 项目进展：关键里程碑
  (7) 融资需求：金额+用途+股权
  (8) 愿景目标：长期价值+社会意义
```

### 来源标注
| 子节 | 来源 | 缺失项 |
|------|------|--------|
| 痛点 | 主档-核心痛点 | 需数据支撑 |
| 解决方案 | 主档-核心功能 | - |
| 核心优势 | 主档-我方优势 | - |
| 市场潜力 | 主档-市场规模 | TAM/SAM/SOM |
| 商业模式 | 主档-商业模式 | 收入测算 |
| 项目进展 | 主档-当前进展 | 需成果证明 |
| 融资需求 | 主档-融资需求 | 金额+用途 |
| 愿景目标 | 主档-社会价值 | 量化描述 |

### 附带输出
```
## 本章PPT提炼
- 页面标题：项目执行摘要
- 核心结论：[一句话概括]
- 3个要点：
  [1] 解决了什么痛点
  [2] 核心竞争优势
  [3] 市场机会与团队实力
- 配图关键词：business pitch, startup, innovation
- 图表建议：核心数据亮点卡片式展示
- 缺失证据清单：
  □ 市场规模数据来源
  □ 竞品对比数据
  □ 用户调研数据
```

## 第二章 项目背景与痛点分析

### 本章目标

  2.2.1 目标用户面临的具体问题
  2.2.2 现有解决方案的不足

2.3 社会需求（红旅/公益项目重点）
  2.3.1 服务国家战略方向
  2.3.2 解决社会问题的价值
```

### 来源标注
| 子节 | 来源 | 缺失项 |
|------|------|--------|
| 行业背景 | 主档-行业背景 | 行业报告 |
| 痛点1 | 主档-痛点1 | 用户调研 |
| 痛点2 | 主档-痛点2 | 用户访谈 |
| 现有方案不足 | 主档-竞品分析 | 竞品功能对比 |

### 附带输出
```

- 核心结论：[目标用户]面临[痛点]，市场规模[X]亿
- 3个要点：
  [1] [痛点量化数据]
  [2] [现有方案不足]
  [3] [市场机会]

- 图表建议：痛点对比柱状图/饼图
