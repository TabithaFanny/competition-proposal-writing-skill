# Rewrite Execution Protocol

> 本协议是 review-rewrite-protocol 的第三步"逐章修订"的具体执行指南。
>
> 核心目的：将"文体约束"（不能写成什么样）转化为"改写动作"（具体该怎么改）。
>
> 使用前提：已熟悉 output-standard.md 的八条基本规则和文体约束，已完成结构化 review。

---

## 一、改写前：识别病灶类型

改写前必须先读完全文，判断原文属于哪种"病"，再匹配对应改写方法。禁止不诊断就直接改。

### 六种典型病灶

| 病灶编号 | 病灶名称 | 症状特征 | 对应改写方法 |
|----------|----------|----------|--------------|
| T1 | 政策罗列型 | 堆砌政策文件、缺乏分析，只写"政策说了什么"不写"对本项目意味着什么" | 五层论证 |
| T2 | 提纲扩写型 | 大量"第一是...第二是...第三是..."，或bold标签式小标题，内容是提纲的展开而非论证 | 拆句扩写 + 去标签化 |
| T3 | 短句堆叠型 | 连续独立短句，缺乏论证展开，读起来像列表或PPT文案 | 合并重组 + 段落衔接 |
| T4 | 论证不足型 | 有论点但缺乏证据，结论跳跃，数据缺乏解释 | 先写厚再收敛 |
| T5 | 展示稿口吻型 | 含meta叙事、元说明、面向评委的提示，或"令人振奋"等宣传腔 | 去meta + 文风收敛 |
| T6 | 缺项目落点型 | 有一般性分析但无对本项目目标、节奏、指标的具体影响 | 补出项目层结论 |

### 病灶处理优先级

```
先处理T5（去meta/宣传腔）
→ 再处理T2（去标签/提纲结构）
→ 再处理T1/T4/T6（补论证/补落点）
→ 最后处理T3（段落衔接）
```

## 二、固定改写动作词表

改写时只能使用以下动词化动作，不得使用"优化"、"加强"、"改进"等模糊表述。

### 允许使用的改写动作

| 动作类别 | 适用场景 | 动作动词 |
|----------|----------|----------|
| 删除类 | 删除标签句、meta句、引导句 | 删除、移除、清去 |
| 合并类 | 合并短句、分点罗列 | 合并、整合、并入 |
| 改写类 | 改变表述方式 | 改写为、重构为、转化为 |
| 补出类 | 补出缺层论证 | 补出、增加、添加 |
| 重组类 | 调整内容顺序 | 前置、后接、重排 |
| 展开类 | 将判断句扩写为论证段 | 展开、延伸、铺陈 |
| 收束类 | 段末或篇末总结 | 收束、归结、点明 |
| 衔接类 | 建立段落间逻辑连接 | 增加承上句、增加过渡句 |

## 三、四栏制问题诊断模板

每发现一个问题点，必须按以下四栏逐项填写：

```
问题点编号：
原句：[摘录具体有问题的句子，不超过一句]
问题类型：[归入T1-T6之一，附上简述]
改写动作：[从第二章固定动作词表中选取，写明做什么]
协议依据：[引用本协议具体条款，说明为什么这样改]
```

### 四栏制示例（标准执行卡格式）

---

**协议执行卡：T2提纲扩写型 + T5展示稿口吻**

原文类型判断：T2（提纲扩写型）+ T5（展示稿口吻）
本段主问题：有措施，但写成了提纲式罗列，缺少解释层和项目落点。
重写目标：改为正式项目书正文体，形成"风险—影响—应对逻辑—措施机制—项目落点"的连续论述。

---

**问题点1**
- **原句**：为降低这一风险，我们采取四项措施。
- **问题类型**：T5 / 标签式引导句
- **改写动作**：删除该句，不保留"提示接下来要分点展开"的功能
- **协议依据**：T5要求删除展示稿口吻和提纲型引导句。引导句的功能是"告诉读者下面要讲什么"，而非"论述内容本身"，必须清除

---

**问题点2**
- **原句**：一是……二是……三是……四是……
- **问题类型**：T2 / 提纲扩写型
- **改写动作**：将并列分点改写为承接式连续段落，用"核心手段—进一步降低阻力—确保进入预算—形成风险对冲"的逻辑顺序串联
- **协议依据**：T2要求把列举式分点改写为连贯的正式段落；将"罗列措施"改为"说明措施之间的内在逻辑"

---

**问题点3**
- **原句**：通过免费试点让社区提前体验产品价值，形成使用习惯，等正式采购时决策周期大幅缩短。
- **问题类型**：T4 / 论证跳跃，缺解释层
- **改写动作**：在该措施后补出"为什么能够缩短决策周期"的机制解释
- **协议依据**：五层论证法，补第三层（解释层），说明措施为何成立、为何有效

---

**问题点4**
- **原句**：如果政府采购周期过长，将直接影响Year2的收入目标。
- **问题类型**：T6 / 缺项目落点
- **改写动作**：将影响具体化，补写为"原计划Year2签约的社区可能推迟到Year3，导致Year2收入大幅缩水"
- **协议依据**：五层论证法，补第四层（项目影响层），把一般性风险转化为本项目年度目标和推进节奏的具体后果

---

**问题点5**
- **原句**：四是建立多层级目标客户池，同时跟进10+个潜在社区，确保即使部分社区采购延迟其他社区的进度不受影响。
- **问题类型**：T3 / 段落衔接不足 + T6 / 缺项目落点
- **改写动作**：在该措施前增加承上句"这一风险意味着我们必须主动缩短采购决策周期"；将"确保不受影响"改写为具有因果逻辑的表述
- **协议依据**：段落衔接规则（每段第一句应承接上段末句）+ 五层论证法

---

**验收标准**
- [ ] 不保留"为降低这一风险，我们采取四项措施"
- [ ] 不保留"一是…二是…三是…四是…"的分点结构
- [ ] 每项措施后有"为什么有效"的机制解释
- [ ] 有对Year2目标的具体影响描述
- [ ] 整体读起来像正式正文，不像答辩提纲

## 四、每段最低生成要求

改写后的每段必须同时满足以下六条：

1. **有明确论点句**：第一句亮出判断，不是现象描述
2. **有解释或证据**：不是只写判断，必须展开"为什么"
3. **有数据或案例**：论点必须有支撑，不能纯概念
4. **有落点**：段末必须有结论或过渡句
5. **无meta残留**：不能有任何面向写作者/读者的提示
6. **无标签残留**：不能有bold标签、numbered list等展示型结构

### 禁止的段落模式

```
❌ 判断句结尾（无论证）
"需求真实。"（没有展开）

❌ 列表罗列（展示型）
"三大核心优势：一、二、三。"

❌ 标语式短句（无论证）
"AI赋能，效率倍增。数据驱动，精准服务。"

❌ 跳跃式论证（缺环节）
"政策支持。技术成熟。市场很大。我们成功了。"（中间缺推导）

❌ 过度展开（论证冗余）
一个论点用500字举例三个同类案例

❌ 标签式引导句
"针对这一问题，我们采取以下措施。"
```

## 五、T1-T6 各病灶改写方法

### T1 政策罗列型 → 五层论证

**症状**：连续3段以上都是政策文件名称和内容摘录；没有"为什么这个政策对本项目重要"的论述。

**改写动作**：展开为五层
- 第一层（判断）：提出结论性观点
- 第二层（证据）：列明政策文件和数据
- 第三层（解释）：政策意味着什么（三个维度展开）
- 第四层（对本项目）：对项目切入时机、预算、决策周期的影响
- 第五层（收束）：归结为本项目的核心竞争优势

### T2 提纲扩写型 → 拆句扩写 + 去标签化

**症状**：大量"第一是...第二是..."或bold标签式结构。

**改写动作**：
1. 识别并删除标签式引导句（如"针对这一问题，我们采取以下措施"）
2. 将分点改为承接式段落
3. 用递进逻辑（不是并列逻辑）串联各项措施
4. 每项措施后补出"为什么有效"

### T3 短句堆叠型 → 合并重组 + 段落衔接

**症状**：连续独立短句，像顺口溜或PPT要点。

**改写动作**：
1. 识别相邻短句的共同主题
2. 合并为包含主语的完整复句
3. 用"——"或因果连词建立句内逻辑
4. 检查段与段之间是否有承上句

### T4 论证不足型 → 先写厚再收敛

**症状**：有论点但缺乏证据，结论跳跃，数据缺乏解释。

**改写动作**：
1. 针对每个判断追问"证据是什么"
2. 补出证据层
3. 再追问"这个证据对本项目意味着什么"
4. 最后收敛：删除冗余，保留核心论证

### T5 展示稿口吻型 → 去meta + 文风收敛

**症状**：含meta叙事、元说明、宣传腔。

**改写动作**：
1. 删除"评委看到..."、"评委会关注..."类meta叙事
2. 删除"本节用于说明..."类元说明
3. 将"令人振奋"、"重磅"等宣传腔改为中性陈述
4. 删除"早发现、早介入"等口号化表达
5. 用逻辑连接词替代"下面"、"首先"、"其次"等机械过渡

### T6 缺项目落点型 → 补出项目层结论

**症状**：有一般性分析但没有落到本项目目标、节奏、指标上。

**改写动作**：
1. 追问"这个判断对项目意味着什么"
2. 补写对年度目标、推进节奏、关键指标的具体影响
3. 用具体数字替代模糊表述

## 六、改写执行顺序（默认工作流）

改写任何章节时，按以下六步执行：

```
第一步：【诊断】通读全文，判断病灶类型（T1-T6）
↓
第二步：【排序】按T5→T2→T1/T4/T6→T3顺序处理
↓
第三步：【执行】按第三章匹配方法，对每个问题点执行"四栏制"改写
↓
第四步：【自检】每段满足"每段最低生成要求"六条
↓
第五步：【通读】全章改完后通读，检查段落衔接
↓
第六步：【输出】直接输出修订稿正文，不加任何meta说明
```
