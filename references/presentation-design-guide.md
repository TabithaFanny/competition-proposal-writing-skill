# 答辩级视觉设计规范 / Presentation Design Guide

> 专门约束：答辩页构图、视觉层级、栅格对齐、图标规则、一页一结论。
> 核心目标：把"会摆信息"升级成"会做展示"。

---

## 核心定位

### 不是网页UI，是答辩页

| 网页UI | 答辩页 |
|--------|--------|
| 滚动浏览 | 3秒定格 |
| 可点击交互 | 静态呈现 |
| 信息密度高 | 一页一结论 |
| 多导航 | 无导航 |
| 模块均等 | 有主次 |

### 答辩页的本质

- **投影环境**：评委远处看，不是近距离读
- **时间压力**：3秒内传达核心
- **记忆点**：评委能记住1个结论就够了
- **视觉冲击**：远观也能分辨主次

## 视觉层级原则

### 第一法则：标题必须有压迫感

```
标题 = 核心结论
字号 = 全页最大
位置 = 顶部或视觉中心
颜色 = 最深/最重
```

**禁止：**
- 标题太小
- 标题和正文差不多大
- 标题颜色太浅

### 第二法则：视觉中心必须唯一

```
视觉中心 = 最想让评委第一眼看到的东西
可以是：
- 一个大数字
- 一个核心词
- 一个关键对比
- 一条路径起点

不能有两个中心同时抢注意力
```

### 第三法则：信息分区必须有主次

```
主区（60%面积）：核心信息
次区（30%面积）：支撑信息
补区（10%面积）：补充说明

不能三个区一样大
```

---

## 一页一结论原则

### 核心规则

每页只讲一个结论，这个结论必须：
1. 是这页最想让评委记住的
2. 通过标题直接可见
3. 通过视觉中心突出

### 常见错误

| 错误 | 问题 | 修正 |
|------|------|------|
| 把多个结论放一页 | 评委记不住 | 拆成多页 |
| 结论藏在正文里 | 评委看不到 | 结论进标题 |
| 结论和支撑信息并列 | 主次不清 | 结论放大/加粗 |

## 构图原则

### 横向构图优先

答辩投影通常是横向16:9或4:3：
- 横向布局适配投影
- 上下构图适合Banner
- 中心放射适合定位图

### 栅格系统

```
8列栅格（适配4:3）
或
12列栅格（适配16:9）

内容对齐栅格线
间距保持一致（8px/16px/24px/32px）
```

### 留白原则

```
不能拥挤
宁可用更少的信息
不用把所有信息塞满

高级留白 = 有意留空引导视线
低级拥挤 = 没有组织好信息
```

---

## 字号层级

```
一级标题（页面主标题）：32-48px
二级标题（模块标题）：18-24px
正文（说明文字）：14-16px
辅助（注释/标签）：10-12px
```

**规则：**
- 一级和二级之间差距至少8px
- 正文和辅助之间差距至少4px
- 颜色深浅辅助表达层级

## 颜色使用

### 主色原则

```
主色 = 本项目强调色
辅色 = 竞品/对比项中性色
背景 = 白/浅灰
文字 = 深灰/黑色

颜色数量不超过3种
```

### 禁用

- 彩虹色
- 高饱和色
- 渐变滥用
- 颜色平均分配

### 禁止的"AI审美"模式

这是最容易出现的视觉陷阱，生成HTML时必须主动避免：

| AI审美模式 | 问题 | 正确做法 |
|------------|------|----------|
| 紫色渐变背景 | 像AI做的 | 科技蓝灰体系 |
| 过度居中布局 | 呆板无重点 | 结构化布局 |
| 统一圆角（16px everywhere） | 单调 | 根据元素类型区分圆角 |
| Inter字体 | 辨识度高但太常见 | 黑体/Heiti/Noto |
| 大量卡片均排布 | 像组件demo | 有主次的分区 |
| 彩虹色配色 | 廉价感 | 克制的主辅色 |
| 居中独占大标题 | 没有视觉张力 | 左对齐+大字号+深色 |

### 设计原则

```
好的答辩页 ≠ 炫酷的效果
好的答辩页 = 3秒能看懂 + 评委能记住

避免：紫色/居中/卡片堆砌/Uniform圆角
追求：结构清晰/主次分明/投影可读
```

## 图标使用规则

### 同一套图标体系

```
风格：线性图标（stroke-based）
粗细：统一stroke-width
圆角：统一border-radius
尺寸：成比例（24/32/48px）
```

### 图标只做辅助

- 图标不能抢主体
- 图标帮助理解，不承载核心信息
- 不用图标装饰

## 元素关系图设计

### 关系图vs卡片排版

**卡片排版（错误）：**
```
┌─────┐ ┌─────┐ ┌─────┐
│  A  │ │  B  │ │  C  │
└─────┘ └─────┘ └─────┘
```
只是并列，没有关系

**关系图（正确）：**
```
       A
      ↗ ↓
     B←──→C

边表示关系：
- A→B：转化
- B→C：协同
```

### 三角定位图结构

```
顶点 = 三个核心要素
边 = 要素间关系
中心 = 核心价值锚点

三者不是并列，而是构成定位体系
```

## 可视化流程（新版）

### 第一步：判断图的目标

回答：
1. 这张图想让评委**记住什么结论**？
2. 是讲**结论**、讲**关系**、讲**差异**、还是讲**路径**？
3. 核心信息是什么？

### 第二步：出低保真结构

**输出格式：**

```
## [图名称] 低保真方案

### 核心结论
[一句话，说出想让评委记住什么]

### 视觉中心
[评委第一眼应该看到什么]

### 页面结构
[用文字描述布局]
- 标题区：
- 主视觉区：
- 辅助区：
- 结论区：

### 元素数量
[几个主要元素，控制在5个以内]

### 版式草图（文字描述）
[描述元素位置关系]

### 设计理由
[为什么这样布局，而不是其他方案]
```

### 第三步：出高保真版本

获得确认后，再输出：
- HTML单页
- Python图表
- 或图片

## 自查清单

生成答辩页后，逐项检查：

```
□ 评委3秒内能说出这页想说什么吗？
□ 标题是核心结论吗？
□ 视觉中心唯一且突出吗？
□ 信息有主次之分吗？
□ 信息量是否过载（>5个主要元素）？
□ 字号层级清晰吗？
□ 颜色不超过3种吗？
□ 留白是"有意"还是"空白"？
□ 像答辩页还是像网页demo？
□ 是在讲故事还是在摆信息？
```

## 常见错误对照

| 错误 | 原因 | 修正 |
|------|------|------|
| 标题太小 | 没理解投影环境 | 标题32px以上 |
| 多个中心 | 没确定唯一重点 | 只留一个视觉中心 |
| 元素均等 | 没有主次意识 | 60/30/10分区 |
| 像网页demo | 用网页思维做答辩 | 3秒定格，投影优先 |
| 塞满信息 | 舍不得删 | 一页一结论 |
| 装饰太多 | 图标喧宾夺主 | 图标只做辅助 |
