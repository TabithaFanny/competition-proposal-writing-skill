# 成稿级长文写作规范 v2.1

> 定义：什么叫真正可交付的比赛申报书正文，而不是提纲、分析卡片、PPT文案、宣传稿。
>
> 核心原则：扩写靠论证维度，不靠文字填充；靠解释"为什么"，不靠情绪词。
>
> **v2.1新增**：拆句扩写规则 · 关键判断段落化 · 段落衔接增强

---

## 两种输出模式

### 模式1：分析稿（便于Review）
- 适合：内部审核、结构调整、数据核对
- 特点：结论先行、要点清晰、便于快速定位问题
- 格式：标题+要点列表+数据表格+简短结论

### 模式2：成稿（正式比赛文书）
- 适合：直接提交、可交付评委
- 特点：连续正文、多维论证、排除法思维、克制文风
- 格式：标准论文风长段落为主，关键处用图表

## 成稿级章节标准

### 长度要求

| 章节类型 | 分析稿字数 | 成稿件字数 | 说明 |
|----------|-----------|-----------|------|
| 执行摘要 | 500-800字 | 800-1200字 | 精炼但完整 |
| 项目背景 | 800-1000字 | 2000-3000字 | 需充分铺垫+论证 |
| 市场分析 | 1000-1500字 | 2500-4000字 | 8个维度+排除法 |
| 产品介绍 | 1000-1500字 | 2500-3500字 | 技术+场景+排除法 |
| 商业模式 | 800-1200字 | 2000-2500字 | 6个维度+盈亏分析 |
| 营销策略 | 800-1200字 | 1500-2000字 | 可执行+阶段路径 |
| 团队介绍 | 600-800字 | 1200-1800字 | 能力矩阵+协作证明 |
| 财务分析 | 800-1200字 | 1500-2000字 | 假设透明+可验证 |
| 风险分析 | 600-800字 | 1200-1800字 | 务实+有应对 |
| 社会价值 | 800-1200字 | 1500-2000字 | 有情怀但克制 |

**关键原则**：成稿件比分析稿多50%-100%，是因为"论证维度增加"，不是因为"句子拉长"。

## 核心：多维论证结构

### 每小节必须有的5层（不是4句）

```
第1层：提出判断
→ 先说结论或观点

第2层：给出证据
→ 数据、案例、事实

第3层：解释证据意味着什么
→ 为什么这个证据支持你的判断

第4层：说明对本项目的影响
→ 这个判断→我们下一步怎么做

第5层：收束成结论
→ 所以我们……
```

### 有效扩写 vs 注水扩写

**有效扩写（增加论证维度）**：
```
原始：需求真实。
扩写：需求真实——300份问卷显示，71.33%社工将重复性问题列为最大痛点，90.33%对AI分类持正面态度。这意味着产品目标用户不仅存在，而且具备技术接受度基础。对于本项目而言，这验证了我们的核心假设，降低了市场教育成本。
```

**注水扩写（句子拉长但没新信息）**：
```
原始：需求真实。
扩写：经过深入的调研分析，我们发现一个令人振奋的事实——广大社区工作者对于智能化工具的引入，普遍持开放欢迎态度，这一发现极大地增强了我们对项目前景的信心。
```

### 多维度举例：目标市场选择

**单维度（不够）**：
```
需求真实 → 结论：选这个市场
```

**多维度（充分）**：
```
维度1-需求真实性：90%支持
维度2-服务匹配性：产品机制能解决
维度3-付费可行性：街道办有预算
维度4-政策适配性：符合基层治理方向
维度5-复制扩张性：长三角模式可推广
→ 结论：五个维度验证，选择合理
```

## 排除法：成熟计划书的标志

### 必须用的场景

| 问题 | 正面论证 | 排除法 |
|------|----------|--------|
| 为什么选A | A的优势 | 为什么不选B/C |
| 为什么用X方案 | X的效果 | 为什么不选Y/Z |
| 为什么现在做 | 时机成熟 | 为什么不是三年前/三年后 |

### 排除法写作模板

```
本项目选择[方案A]，而非[备选B/C]，基于以下判断：

第一，[排除B]……因此暂不选择B。
第二，[排除C]……因此暂不选择C。
第三，[选择A的独特优势]……这使得A是最优选择。

这种系统性筛选，确保每个战略选择都经过深思熟虑。
```

## 段落密度要求

### 禁止写法

**提纲罗列式**：
```
## 市场规模
一、TAM约600亿
二、SAM约3亿
三、……
```

**表格堆砌式**：
```
市场规模数据如下表：[大表格] 结论：市场很大。
```

**PPT文案式**：
```
【大数字】600亿
【要点1】60万社区
【要点2】年增15%
```

**跳跃论证式**：
```
有60万个社区。（数据）
市场很热。（观点）
所以我们做这个。（跳跃）
```

### 正确段落结构

```
## 4.2 市场规模

第一层（判断）：本项目面向的是一个"大市场、小切入"的格局。

第二层（证据）：全国60万个城乡社区，若按年均5万元数字化投入计算，潜在市场规模约600亿元。

第三层（解释）：然而，600亿是潜在市场，不是可服务市场。从潜在到可服务，需要跨越数字化基础、支付意愿、产品成熟度三重门槛。

第四层（对本项目）：这意味着我们不能贪大求全，而必须选择数字化基础好、支付意愿明确、产品能力匹配的细分市场切入。

第五层（结论）：基于此，我们将SAM估算为3亿元，SOM第三年约30万元。
```

## 过渡句规范

### 必须有过渡的位置

| 位置 | 类型 | 示例 |
|------|------|------|
| 大节之间 | 承上启下 | "在明确市场机会后，我们进一步分析竞争格局" |
| 小节开头 | 背景引入 | "然而，机会存在不等于能顺利切入" |
| 观点转折 | 排除法 | "为什么不选B/C，而选择A" |
| 数据解读 | 意义说明 | "换言之，这意味着……" |

### 过渡句模板

**承上启下**：
- "在上述分析基础上，我们进一步来看……"
- "综合以上讨论，本节聚焦……"
- "上述机会分析，为理解竞争格局提供背景"

**背景引入**：
- "值得注意的是……"
- "需要特别说明的是……"
- "从实践层面看……"

**排除转折**：
- "本项目选择A而非B/C，基于以下判断……"
- "如果不选A而选B，会面临……问题"

**意义解读**：
- "换言之……"
- "也就是说……"

```
□ 节与节有过渡
□ 维与维有衔接
□ 无突兀跳跃

### 数据质量
□ 有来源标注
□ 有解释说明

### 文风克制
□ 无宣传腔（无"令人振奋"等）
□ 无空洞套话（无"前景广阔"等）

□ 评委可从头读到尾
□ 逻辑连贯
□ 读后能记住核心观点
```

## 拆句扩写规则（v2.1新增）

### 核心原则

**一个判断句 → 一个最小论证段落**

每个独立判断必须展开为完整段落，不能用句号切断跳跃到下一个判断。

### 拆句扩写模板

**原始判断**：
```
需求真实。
```

**拆句扩写后**：
```
需求真实——这一判断基于对300份一线问卷的统计分析。数据显示，71.33%的社区工作者将"重复性问题处理"列为最大工作痛点，92%认为"高风险问题预警"是当前最迫切需要解决的问题。换言之，一线社工不仅面临效率瓶颈，更面临风险识别能力不足的结构性问题。对于本项目而言，这意味着我们的目标用户不仅存在，而且有明确付费意愿和紧迫感。
```

### 三步拆句法

```
第一步：识别判断句
   → 找出"结论性判断"（是/不是/因为/所以/应该）

第二步：追问"为什么"
   → 这个判断的证据是什么？
   → 这个证据意味着什么？
   → 对本项目有什么影响？

第三步：写成完整段落
   → 判断 + 证据 + 解释 + 项目意义 + 小结
```

### 常见错误：判断句罗列

**错误写法**：
```
需求真实。痛点明确。产品有价值。市场空间大。团队能力强。所以我们做这个。
```

**正确写法**：
```
需求真实——[展开论证]……这意味着[结论1]。
痛点明确——[展开论证]……这意味着[结论2]。
产品有价值——[展开论证]……这意味着[结论3]。
市场空间大——[展开论证]……这意味着[结论4]。
团队能力强——[展开论证]……这意味着[结论5]。
综合以上，我们决定做这个项目。
```

## 关键判断段落化（v2.1新增）

### 什么叫"关键判断"

凡涉及以下类型的判断，必须展开为独立段落：
- 市场规模结论（"这是一个X亿的市场"）
- 竞争优势声明（"我们的差异化在于X"）
- 技术路线选择（"我们选择A而非B"）
- 商业模式核心假设（"我们的盈利来自X"）
- 团队能力核心判断（"我们具备X能力"）

### 关键判断段落标准结构

```
【判断标题】（可省略标题，但内容必须包含）

第一层-明确判断：我们的核心判断是[具体内容]。

第二层-证据支撑：支撑这一判断的证据包括——1.[数据A]；2.[数据B]；3.[案例C]。

第三层-逻辑解释：这三个证据共同指向[结论]，因为[因果链条]。

第四层-对本项目意义：这一判断意味着我们的[具体策略/方向]是合理的/必要的/可行的。

第五层-收束：因此，[重申判断]。
```

### 示例：市场规模判断段落化

```
全国社区治理数字化市场约300亿元。

【证据层】支撑这一判断的数据包括：全国城市社区约10万个，若按平均年数字化投入30万元计算，市场总量约300亿元。其中，沿海发达城市社区约3万个，年均投入可达50万元，市场规模约150亿元。

【解释层】300亿的市场总量看似庞大，但必须注意两个结构性问题。一是当前数字化渗透率不足20%，实际可服务市场约60亿元；二是社区数字化投入以政府专项资金为主，支付决策周期较长。

【项目意义层】这意味着本项目在切入期应聚焦数字化渗透率较高的沿海发达城市社区，降低市场教育成本和回款周期。

【收束层】综合以上，我们将目标市场（SAM）定为5亿元，可服务市场（SOM）第三年约为500万元。
```

## 段落衔接增强（v2.1新增）

### 衔接失败的三种类型

| 类型 | 症状 | 解决方案 |
|------|------|----------|
| 跳跃式 | 上段结论与下段开头无关联 | 添加承上句 |
| 断裂式 | 两段之间逻辑断层 | 添加过渡段 |
| 重复式 | 下段重复上段内容 | 删除重复，直接推进 |

### 衔接句模板库

**承上启下**：
- "上述分析表明[结论]，基于此，我们进一步来看……"
- "综合以上讨论，[结论]是明确的。接下来我们需要回答的问题是……"
- "在明确[结论]之后，本节进一步论证……"

**问题导向**：
- "然而，[结论]只是必要条件，充分条件还需要满足……"
- "需要回答的关键问题是：是什么决定了[核心变量]？"
- "基于以上论证，一个核心问题浮现出来：如何……"

**递进式**：
- "不仅如此，[结论]的背后还有更深层的原因……"
- "进一步看，[看似矛盾的现象]实际上反映了……"
- "更重要的是，[推进到下一个层次]"

**对照式**：
- "与A路径不同，我们选择B路径，因为……"
- "表面上看[现象X]，但实质上是[本质Y]"
- "一种可能是[方案A]，另一种可能是[方案B]，我们选择后者，因为……"

### 段落衔接自检

```
完成一个章节后，通读检查：
□ 每个段落的第一句是否承接了上一段？
□ 章节开头是否建立了本章要回答的核心问题？
□ 章节结尾是否给出了明确的结论或过渡到下一章？
□ 段落之间是否有断层或重复？
```

## 先写厚再收敛（v2.1新增原则）

### 操作流程

```
阶段1：写厚（默认模式）
   → 按5层论证展开每个判断
   → 每个维度充分展开
   → 允许冗余和重复

阶段2：收敛检查
   → 识别重复论述
   → 合并同类项
   → 删除不服务核心观点的展开
   → 确保每个判断只出现一次

收敛标准：
□ 无重复论证（同一观点只论证一次）
□ 无冗余例子（每个证据只用一个）
□ 无过度展开（论证厚度与论点重要性匹配）
```

### 收敛自检问题

```
拿到初稿后问：
1. 这个段落的核心句是什么？删掉其他句子还成立吗？
2. 这个证据除了支持这个论点，还支持其他论点吗？如果只支持一个，删。
3. 这段文字评委读完会记住什么？如果什么都记不住，删。
4. 这张表/这张图评委看完还会看正文吗？如果不会，保留文字。
```
