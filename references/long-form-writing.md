# 成稿级长文写作规范 v2.0

> 定义：什么叫真正可交付的比赛申报书正文，而不是提纲、分析卡片、PPT文案、宣传稿。
>
> 核心原则：扩写靠论证维度，不靠文字填充；靠解释"为什么"，不靠情绪词。

---

## 两种输出模式

### 模式1：分析稿（便于Review）
- 适合：内部审核、结构调整、数据核对
- 特点：结论先行、要点清晰、便于快速定位问题
- 格式：标题+要点列表+数据表格+简短结论

### 模式2：成稿（正式比赛文书）
- 适合：直接提交、可交付评委
- 特点：连续正文、多维论证、排除法思维、克制文风
- 格式：标准论文风长段落为主，关键处用图表

---

## 成稿级章节标准

### 长度要求

| 章节类型 | 分析稿字数 | 成稿件字数 | 说明 |
|----------|-----------|-----------|------|
| 执行摘要 | 500-800字 | 800-1200字 | 精炼但完整 |
| 项目背景 | 800-1000字 | 2000-3000字 | 需充分铺垫+论证 |
| 市场分析 | 1000-1500字 | 2500-4000字 | 8个维度+排除法 |
| 产品介绍 | 1000-1500字 | 2500-3500字 | 技术+场景+排除法 |
| 商业模式 | 800-1200字 | 2000-2500字 | 6个维度+盈亏分析 |
| 营销策略 | 800-1200字 | 1500-2000字 | 可执行+阶段路径 |
| 团队介绍 | 600-800字 | 1200-1800字 | 能力矩阵+协作证明 |
| 财务分析 | 800-1200字 | 1500-2000字 | 假设透明+可验证 |
| 风险分析 | 600-800字 | 1200-1800字 | 务实+有应对 |
| 社会价值 | 800-1200字 | 1500-2000字 | 有情怀但克制 |

**关键原则**：成稿件比分析稿多50%-100%，是因为"论证维度增加"，不是因为"句子拉长"。

## 核心：多维论证结构

### 每小节必须有的5层（不是4句）

```
第1层：提出判断
→ 先说结论或观点

第2层：给出证据
→ 数据、案例、事实

第3层：解释证据意味着什么
→ 为什么这个证据支持你的判断

第4层：说明对本项目的影响
→ 这个判断→我们下一步怎么做

第5层：收束成结论
→ 所以我们……
```

### 有效扩写 vs 注水扩写

**有效扩写（增加论证维度）**：
```
原始：需求真实。
扩写：需求真实——300份问卷显示，71.33%社工将重复性问题列为最大痛点，90.33%对AI分类持正面态度。这意味着产品目标用户不仅存在，而且具备技术接受度基础。对于本项目而言，这验证了我们的核心假设，降低了市场教育成本。
```

**注水扩写（句子拉长但没新信息）**：
```
原始：需求真实。
扩写：经过深入的调研分析，我们发现一个令人振奋的事实——广大社区工作者对于智能化工具的引入，普遍持开放欢迎态度，这一发现极大地增强了我们对项目前景的信心。
```

### 多维度举例：目标市场选择

**单维度（不够）**：
```
需求真实 → 结论：选这个市场
```

**多维度（充分）**：
```
维度1-需求真实性：90%支持
维度2-服务匹配性：产品机制能解决
维度3-付费可行性：街道办有预算
维度4-政策适配性：符合基层治理方向
维度5-复制扩张性：长三角模式可推广
→ 结论：五个维度验证，选择合理
```

## 排除法：成熟计划书的标志

### 必须用的场景

| 问题 | 正面论证 | 排除法 |
|------|----------|--------|
| 为什么选A | A的优势 | 为什么不选B/C |
| 为什么用X方案 | X的效果 | 为什么不选Y/Z |
| 为什么现在做 | 时机成熟 | 为什么不是三年前/三年后 |

### 排除法写作模板

```
本项目选择[方案A]，而非[备选B/C]，基于以下判断：

第一，[排除B]……因此暂不选择B。
第二，[排除C]……因此暂不选择C。
第三，[选择A的独特优势]……这使得A是最优选择。

这种系统性筛选，确保每个战略选择都经过深思熟虑。
```

## 禁用词清单（修辞过度）

### 宣传腔禁用词

| 禁用 | 替换 |
|------|------|
| 令人振奋 | （直接陈述） |
| 极大/显著增强 | 显著提升/验证 |
| 广大社区 | 一线社工 |
| 颠覆性/革命性 | 创新性/实质性 |
| 绝对领先/无与伦比 | （删除或改写） |
| 精准击中 | 针对性解决 |
| 绝佳试验田 | 合适的试点区域 |
| 坚实起点 | 良好的起步 |
| 一腔热血/热血沸腾 | （禁止） |
| 碉堡了/太强了/牛 | （禁止） |
| 闭门造车 | 脱离实际 |
| 极具商业价值 | 具有商业可行性 |

### 空洞套话禁用

| 禁用 | 替换 |
|------|------|
| 前景广阔 | （用具体数据替代） |
| 市场潜力巨大 | 市场规模X亿元 |
| 深受用户喜爱 | 90%用户表示满意 |
| 响应国家号召 | 符合[具体政策名称]要求 |
| 具有重大意义 | [量化描述] |

## 段落密度要求

### 禁止写法

**提纲罗列式**：
```
## 市场规模
一、TAM约600亿
二、SAM约3亿
三、……
```

**表格堆砌式**：
```
市场规模数据如下表：[大表格] 结论：市场很大。
```

**PPT文案式**：
```
【大数字】600亿
【要点1】60万社区
【要点2】年增15%
```

**跳跃论证式**：
```
有60万个社区。（数据）
市场很热。（观点）
所以我们做这个。（跳跃）
```

### 正确段落结构

```
## 4.2 市场规模

第一层（判断）：本项目面向的是一个"大市场、小切入"的格局。

第二层（证据）：全国60万个城乡社区，若按年均5万元数字化投入计算，潜在市场规模约600亿元。

第三层（解释）：然而，600亿是潜在市场，不是可服务市场。从潜在到可服务，需要跨越数字化基础、支付意愿、产品成熟度三重门槛。

第四层（对本项目）：这意味着我们不能贪大求全，而必须选择数字化基础好、支付意愿明确、产品能力匹配的细分市场切入。

第五层（结论）：基于此，我们将SAM估算为3亿元，SOM第三年约30万元。
```

## 过渡句规范

### 必须有过渡的位置

| 位置 | 类型 | 示例 |
|------|------|------|
| 大节之间 | 承上启下 | "在明确市场机会后，我们进一步分析竞争格局" |
| 小节开头 | 背景引入 | "然而，机会存在不等于能顺利切入" |
| 观点转折 | 排除法 | "为什么不选B/C，而选择A" |
| 数据解读 | 意义说明 | "换言之，这意味着……" |

### 过渡句模板

**承上启下**：
- "在上述分析基础上，我们进一步来看……"
- "综合以上讨论，本节聚焦……"
- "上述机会分析，为理解竞争格局提供背景"

**背景引入**：
- "值得注意的是……"
- "需要特别说明的是……"
- "从实践层面看……"

**排除转折**：
- "本项目选择A而非B/C，基于以下判断……"
- "如果不选A而选B，会面临……问题"

**意义解读**：
- "换言之……"
- "也就是说……"

## 成稿自检清单

```
## 成稿质量自检

### 论证维度
□ 每节有5层（判断→证据→解释→项目影响→结论）
□ 不是一句判断走到底，而是有充分论证
□ 有至少2个横向维度交叉验证
□ 必要时用了排除法

### 段落密度
□ 没有"一逗到底"流水账
□ 每段有明确核心句
□ 段落长度适中（5-10句）

### 过渡衔接
□ 节与节有过渡
□ 维与维有衔接
□ 无突兀跳跃

### 数据质量
□ 有来源标注
□ 有解释说明
□ 有对本项目意义

### 文风克制
□ 无宣传腔（无"令人振奋"等）
□ 无空洞套话（无"前景广阔"等）
□ 无跳跃论证
□ 无自夸感

### 可读性
□ 评委可从头读到尾
□ 逻辑连贯
□ 读后能记住核心观点
```
